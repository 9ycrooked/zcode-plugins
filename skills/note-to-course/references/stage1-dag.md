# 阶段 1:萃取知识点 DAG

本文件在 SKILL.md 路由到阶段 1 时按需加载。这是整个管线的地基,必须人工审核。

## 目标

从一篇(或多篇)思源笔记中,萃取出这个主题的知识点有向无环图(DAG)。
这是后续所有阶段的依据——`objective` 字段错了,后面全错。

## 输入

- **笔记原文**:用 `document.get_doc` 读取目标笔记的 markdown 全文
- **笔记本目录**:用 `document.list_tree` 获取章节标题列表,用于标注跨章节依赖

## 知识点粒度三约束

每个知识点必须**同时**满足:

1. **可诊断**——能写出一道题区分"会"和"不会"。写不出诊断题 = 粒度不对
2. **可讲清**——一份内容包能讲清楚(讲解+代码+trace+练习)
3. **可学完**——概念型 15-25 分钟,逻辑型 25-40 分钟

判断不满足时:
- 太大(一道题测不了) → 拆分
- 太小(讲不了一节课) → 合并到父知识点

## Prompt 指引

分析笔记时按这个思路:

1. **识别原子概念**:笔记里哪些是独立的、值得单独教的概念?
2. **标注依赖**:概念 A 理解是否依赖概念 B?依赖则 `depends_on` 指向 B
3. **检查 DAG 合法性**:依赖关系不能成环
4. **估算时长**:按内容复杂度给 5-60 分钟
5. **写 objective**:用一句话精确描述"学完能做什么",这是后续阶段的唯一依据

## 输出格式(dag.json)

```json
{
  "topic": "Rust 所有权",
  "prerequisites": ["变量与类型", "函数基础"],
  "knowledge_points": [
    {
      "id": "move_semantics",
      "title": "Move 语义",
      "objective": "理解值的所有权只能有一个持有者,赋值后原变量失效",
      "depends_on": [],
      "estimated_minutes": 20
    },
    {
      "id": "immutable_borrow",
      "title": "不可变借用 &T",
      "objective": "理解多个只读引用可以共存,借用不转移所有权",
      "depends_on": ["move_semantics"],
      "estimated_minutes": 20
    }
  ]
}
```

字段约束(见 schema.json):
- `id`:snake_case,如 `move_semantics`
- `depends_on`:只能引用同主题内已定义的 `id`
- `objective`:必须可执行(后续阶段照它生成代码/trace/练习)

## 人工审核要点

向用户展示 DAG 后,逐项确认:

1. **粒度**:每个知识点是否满足三约束?有没有该拆没拆/该合没合的?
2. **依赖**:`depends_on` 是否准确?有没有遗漏的隐含前置?
3. **objective** 是否精确?——这是最重要的。模糊的 objective 会导致后面代码生成跑偏
4. **拓扑序**:knowledge_points 数组是否按拓扑顺序排列(被依赖的在前)?

## 完成判定

- DAG 通过人工审核
- 拓扑序验证通过(无环)
- 每个 objective 都是一句可执行的教学目标

审核通过后,才能进入阶段 2-4。
