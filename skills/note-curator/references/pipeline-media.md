# 管线：视频 / 音频导入

本文件在 SKILL.md 路由到媒体导入时按需加载。

## 现状

思源 MCP 无法直接解析视频/音频内容。本管线采用"先记录元数据，后深入"的策略。

## 工作流

1. **提取元数据**：从 URL 或文件名获取标题、描述等信息
2. **写入文档**：
   ```
   # [视频/音频标题的核心知识点]

   [从标题、描述、可获取的信息中提取的关键信息]

   ⏳ 待深入：需要实际观看/收听后补充详细笔记

   ---

   - 来源：[URL 或文件路径]
   - 类型：[课程 / 演讲 / 播客 / 教程 / 会议录音]
   - 关联：[[相关知识文档]]
   - 标签：#topic
   - 状态：待深入
   ```
3. **建链 + 归档**：同纯文本管线

## 后续深入

用户看完视频后，可再触发"笔记再加工"管线，将"待深入"标记更新为正式笔记。

## 如果有字幕文件

- 用户如果提供 `.srt` / `.vtt` 字幕文件，当作纯文本处理（按 `pipeline-text.md`）
- 字幕文本通常很长，需按时间分段原子化，提取每段的独立知识点
