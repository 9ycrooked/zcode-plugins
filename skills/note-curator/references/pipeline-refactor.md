# 管线：笔记再加工

本文件在 SKILL.md 路由到"整理/优化笔记"时按需加载。

## 触发场景

用户说"整理一下我的笔记"、"这些笔记有点乱"、"帮我优化一下"、"重构知识库"等。

## 工作流

1. **扫描**：通过 `search.fulltext` 找出目标范围内的文档
   - 用户未指定范围 → 询问：全库 / 指定笔记本 / 指定文档 / 最近添加的
2. **评估**：检查每条笔记的质量
   - 是否原子化（一条笔记 = 一个知识点）
   - 标题是否是可检索的命题句
   - 是否有来源标注
   - 是否有双向链接连接到其他知识
3. **重组**：
   - 拆分过大的笔记为原子笔记
   - 合并内容重复的笔记
   - 重写非命题句标题
   - 补充缺失的来源和标签
4. **建网**：在知识点之间补充双向链接，建立知识图谱
   - 用 `search.get_backlinks` 检查现有连接
   - 主动搜索相关主题文档，补充 `[[双链]]`
5. **清理**：将已归档/过时的笔记移到 Archives 笔记本
   - 用 `document.move` 移动文档

## 操作工具

| 操作 | 工具 |
|------|------|
| 搜索笔记 | `search.fulltext` |
| 读取文档内容 | `document.get_doc` |
| 更新文档/块 | `block.update` / `block.replace` |
| 移动文档 | `document.move` |
| 重命名文档 | `document.rename` |
| 设置标签/属性 | `document.set_attr` / `tag.rename` |

## 输出摘要

整理完成后，向用户报告：
- 扫描了多少条笔记
- 拆分/合并/重命名/移动各多少条
- 新增了多少个双向链接
- 哪些笔记标记为"待深入"
