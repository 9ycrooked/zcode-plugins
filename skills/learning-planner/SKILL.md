用户想让你帮忙规划一条学习路径。你需要做以下事情：

## 第一步：理解用户需求

使用 `AskUserQuestion` 工具向用户提问，不要自己猜测。

1. 用户说了想学什么（多个技术栈）
2. 用 `AskUserQuestion` 询问用户对每个技术栈的掌握程度（零基础/有基础/已掌握）
3. 用 `AskUserQuestion` 确认学习目标（短期/长期，用于工作还是兴趣）

## 第二步：设计学习路径

按依赖关系排序，设计分阶段路线。示例格式：

```
第一阶段 ──→ 第二阶段 ──→ 第三阶段
A基础        B框架         C进阶
```

对于每个阶段标注预计耗时。

## 第三步：在思源创建项目文档

使用 `fs(action="write", path="/1-学习区/<项目名>", markdown="...")` 为每个阶段创建独立的项目文档。

文档内容结构：
- 学习目标（一句话）
- 学习内容（要点列表）
- 预计耗时
- 前置条件

同时创建/更新 `/1-学习区/学习路径图` 文档，用 ASCII 路线图展示完整路径。

## 第四步：录入学习项目看板数据库

数据库 AV ID：`20260704163235-rvkdhnk`

1. 使用 `av(action="add_rows", avID="20260704163235-rvkdhnk", blockIDs=["<新文档ID>"])` 添加每个项目
2. 使用 `av(action="render", id="20260704163235-rvkdhnk")` 获取新行的 rowID
3. 使用 `av(action="set_cells", avID="20260704163235-rvkdhnk", cells=[...])` 设置属性：

```json
{
  "rowID": "<新行ID>",
  "columnID": "20260704163249-5hkv0lz",  // 状态列
  "valueType": "select",
  "option": "学习中"   // 第一个项目or"待启动"
}
```

列的 keyID 对照：
- 状态：`20260704163249-5hkv0lz`（选项：学习中/待启动/待复习/已完成）
- 学习目标：`20260704163250-ajsn1q9`
- 优先级：`20260704163250-c7wqobn`（选项：🔴高/🟡中）
- 截止日期：`20260704163250-5d5nft0`

## 第五步：更新路径图文档

用 `fs(action="write", path="/1-学习区/学习路径图", overwrite=true)` 更新总览路线图，包含所有阶段，并用 `[[项目名]]` 链接到各项目文档。

## 状态管理规则

- 当前正在学的阶段 → `学习中`
- 后续阶段 → `待启动`
- 前置条件未完成的 → `待启动`
- 用户已掌握的 → 文档标注 ✅，不入数据库

## 示例

用户说："我想学前端动画，HTML5 CSS3有基础，JS会一点"

预期输出：
1. 创建文档：HTML5+CSS3（已掌握标注✅）、JavaScript基础、Anime.js、GSAP
2. 数据库录入 JS基础(学习中)、Anime.js(待启动)、GSAP(待启动)
3. 路径图：JS基础 → Anime.js → GSAP
4. 用 [[ 链接关联各文档
