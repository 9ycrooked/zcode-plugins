# 思源笔记操作备忘

本文件供 skill 在需要时参考，不直接呈现给用户。

## 知识文档模板

```markdown
# [核心命题 — 可检索的句子]

[正文段落：用自己的话表述，2-5 段]

> 原文参考（如有）：引用原文关键句

---

**元数据**（用思源文档属性或属性视图管理）

| 字段 | 值 |
|------|-----|
| 来源类型 | 书籍 / 文章 / 视频 / 对话 / 实践 |
| 来源 | URL 或标题 |
| 知识类型 | 概念 / 方法 / 案例 / 引用 / 术语 / 工具 |
| 状态 | 新建 / 已关联 / 待深入 / 已归档 |
| 创建日期 | 自动 |
| 关联项目 | [[项目文档]] |
```

## PARA 笔记本结构建议

```
📚 知识库
├── 📁 Projects        ← 当前进行中的项目相关笔记
├── 📁 Areas           ← 长期关注的领域（前端、AI、写作...）
├── 📁 Resources       ← 通用知识、参考资料、灵感
├── 📁 Archives        ← 已完成的项目、不再活跃的知识
└── 📋 知识索引        ← 索引页 / MOC（Map of Content）
```

## 属性视图 Schema 建议

用思源的属性视图（AV）管理知识元数据，建议字段：

- **标题**（text）— 文档标题，即命题句
- **来源类型**（select）— 书籍/文章/视频/对话/实践
- **来源**（text）— URL 或出处
- **知识类型**（select）— 概念/方法/案例/引用/术语/工具
- **状态**（select）— 新建/已关联/待深入/已归档
- **关联项目**（relation）— 关联的项目文档
- **创建日期**（date）

这样可以通过 AV 的过滤/排序/分组快速检索和复习。
