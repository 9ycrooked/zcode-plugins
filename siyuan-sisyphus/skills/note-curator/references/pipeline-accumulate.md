# 管线：持续积累

本文件在 SKILL.md 路由到日常积累 / 回顾复习时按需加载。

## 触发场景

- 日常对话中产生了值得记录的知识点
- 用户主动说"帮我回顾一下"、"复习"、"看看最近记了什么"

## 子流程 A：日常捕获

1. **识别**：在日常对话中识别值得记录的知识点（顿悟、纠正认知、可复用方法等）
2. **提示**：主动询问用户"这条知识要记入知识库吗？"
3. **提炼**：获确认后，将对话中的知识提炼为原子笔记
   - 按 `pipeline-text.md` 的格式写入
4. **关联**：连接到已有知识网络

## 子流程 B：回顾复习

用户触发回顾时，从知识库中抽取并呈现：

1. **最近新增**：用 `search.fulltext` 按 `updated_desc` 排序，取最近 7 天的笔记
   - 呈现标题列表 + 一句话摘要，帮助快速回忆
2. **孤立笔记**：检查反向链接数为 0 的笔记
   - 用 `search.get_backlinks` 逐条检查
   - 推荐补充关联：为孤立笔记找到 1-2 个相关文档
3. **待深入笔记**：搜索标记为"待深入"的笔记
   - 用 `search.fulltext` 搜 `待深入` 关键词
   - 推荐用户深入阅读或补充内容

## 回顾呈现格式

```
📚 知识库回顾（最近 7 天）

🆕 新增笔记（5 条）
- [[费曼学习法的本质]] — 输出倒逼输入
- [[DRY 原则的局限]] — 过度抽象反而增加认知负担
- ...

🏝 孤立笔记（2 条，建议补充关联）
- [[心流状态的平衡条件]]
- ...

⏳ 待深入（3 条）
- [[Transformer 自注意力机制]]
- ...
```

## 操作工具

| 操作 | 工具 |
|------|------|
| 按时间搜索 | `search.fulltext` (sortBy: updated_desc) |
| 检查反向链接 | `search.get_backlinks` |
| 按关键词搜索 | `search.fulltext` (query: "待深入") |
| 创建笔记 | `document.create` + `pipeline-text.md` 流程 |
